Getting started with development

Getting started with development of SlowCheetah is pretty easy. You will need the following to get started:
  • Visual Studio 2010
  • Visual Studio 2010 SDK

Components of SlowCheetah

SlowCheetah is pretty simple, it mainly consists of the following componetns.
  • Visual Studio plugin for UI support
  • MSBuild .targets file to support transforms during build
  • Unit tests

Here is an overview of how SlowCheetah works.

For supported projects (any project with an extension of .csproj/.vbproj/.fsproj) a new context menu will appear for XML files, Add Transform. When a user invokes this command the following will occur:

  1. Files needed to support transformations will be dropped at %localappdata%\Microsoft\MSBuild\SlowCheetah\v1\
  2. The user will be prompted to approve changes to the project file
  3. The project file will be updated to include an Import to the .targets file
  4. Transform files are added to the project as child nodes of the file

Note: Step 1 in the above actually occurs when the package is loaded (i.e. when a project is first loaded in VS.) After this when a build is performed in VS or in the command line the XML files with transforms will be transformed. If a user invokes Add Transform after the import has been added the command will simply add the transform files. The files which get installed include the following.
  • Install-Manifest.xml - This is a file which contains; the version of installed files and the files that are included
  • SlowCheetah.tasks.dll - This is the file which contains the TransformXml task
  • SlowCheetah.Transforms.targets - This file is imported into the user's project file. It contains all the logic to hookup the transformations.
The Install-Manifest.xml file contains an attribute on the root element version. Each time that the SlowCheetah package is loaded in VS it will compare its version against the Install-Manifest.xml. If the installed files are out of date (i.e. the running version has a higher value for version) then the new files will be installed. Below are important files you will mostly be editing in the repository.
  • SlowCheetah.sln - The solution file you will open with VS 2010
  • SlowCheetah\SlowCheetahPackage.cs - most of the code for the VS UI support
  • SlowCheetah\Resources\Install\SlowCheetah.Transform.targets - MSBuild .targets file

Things to keep in mind during development

Here are a few things to keep in mind while developing SlowCheetah.

When you are bulding the SlowCheetah.sln file there is a target which will be invoked that will automatically copy the required files to %localappdata%\Microsoft\MSBuild\SlowCheetah\v1\, if the files are in use you will receive an error during the build process.

In order to resolve that you should close all instances of VS which may have the assembly loaded (i.e. any VS instance which you were testing transforms with). If you are using VS 2012 after you close VS the assemblies may still be locked by msbuild.exe. In this case you may have to kill msbuild.exe using the Task Manager. VS 2012 msbuild.exe is re-used so this behavior is by design from the MSBuild team.

In order to build a new .vsix to test out just build the project in VS (either in debug or release mode) and then grab the .vsix in the output folder (i.e. bin\debug\)

In order to submit your changes back to the source repo send a pull request to the project. It will be processed by a coordinator.
